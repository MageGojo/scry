# 设计 · 渗透/利用能力可行性分析 + 内置漏洞扫描增强

> 用户需求:把「综合渗透框架(Metasploit / Cobalt Strike / Sliver)」与「Web 漏洞专项
> (sqlmap / XSStrike / dalfox / Nikto / Wapiti)」加进 scry,**能内置就内置**,不能内置要
> 诚实说清楚,并**挑一个合适的先落地**。本文档是这次的分析结论 + 落地设计。

## 1. 结论先行:哪些能内置,哪些不能

scry 的内核身份 = **TLS 终止式 HTTP(S) MITM 代理 + Burp 式工作台**(Proxy / Repeater /
Intruder / Scanner / Decoder / Comparer / Spider / 扩展)。判断「能不能内置」就看它是否落在
这条「请求 → 响应 → 判定 / 改写」的主轴上。

### 1.1 三、综合渗透框架 / C2(❌ 不适合内置)
| 工具 | 语言/形态 | 能否内置 | 原因(诚实) |
|---|---|---|---|
| **Metasploit** | Ruby,GB 级框架 | ❌ | 独立巨型生态,只能当 `msfrpcd` 的 RPC 客户端;且核心是**主机/网络漏洞利用 + shell/Meterpreter 后渗透**,与「Web 流量分析」不是一条轴。 |
| **Cobalt Strike** | 商业闭源,$3k+/年 | ❌ | 闭源 + 商业授权,**法律上不可分发 / 集成**。无解。 |
| **Sliver** | Go,开源 C2 | ❌(可外接) | 比 CS 干净,但仍是独立 C2 生态(gRPC server/client),只能「外部进程 + gRPC 对接」,本质是后渗透/上线管理,偏离 scry 主轴。 |

**判断**:C2 / 漏洞利用框架**做不到真正内置**,强行塞进来只会变成「启动器/RPC 壳」,是范围蔓延。
受挫点如实说明:不是不想做,是它们与 scry「Web MITM 工作台」根本不是同一类产品。
(将来若真要,只能做成「扩展系统里的外部工具桥」,单列、非本次。)

### 1.2 四、Web 漏洞专项(✅ 适合内置,且正中 scry 主轴)
| 工具 | 本质 | scry 现状 | 内置路径 |
|---|---|---|---|
| **sqlmap** | 深度 SQLi | 已有 error-based SQLi 探测(`active.rs`) | 原生 Rust 可做子集(报错/布尔/时间盲注),深化即可 |
| **dalfox / XSStrike** | XSS | 已有反射 XSS 探测 | 同上,扩展注入上下文 |
| **Nikto / Wapiti** | **已知危险路径/文件探测 + 配置错误 + 参数模糊** | 被动规则 + 主动参数注入已具雏形 | **完全可原生内置**:一张高危路径库 + 「请求→匹配」即可 |

这些工具全是「构造 HTTP 请求 → 匹配响应特征」,**与 scry 的 `scry_scan` + `replay` + `crawler`
模型同构**,纯 Rust 内置即可,零外部运行时、无授权问题,还契合「零环境出包」目标。

## 2. 本次挑的「一个合适的」:Nikto 式内置敏感文件/路径扫描

**为什么是它**:在「能内置」的 Web 类里,Nikto 的精髓(已知危险文件/目录探测)是
**最高 ROI、最纯粹可内置、最对 scry 胃口**的一块——也是 scry 目前**唯一缺**的主动能力
(已有参数注入 = Wapiti/sqlmap 的另一半;内容发现 = 这一半)。补上后,scry 主动扫描 ≈
Nikto(内容发现) + Wapiti(参数模糊) 的覆盖。

### 2.1 内核(`scry_scan::discovery`,纯函数 + 单测)
- **高危路径库 `PATHS`**:VCS(`.git/HEAD`/`.git/config`/`.svn/wc.db`/`.hg`)、环境凭据
  (`.env`/`.aws/credentials`/`.npmrc`/`.htpasswd`)、备份与库导出(`backup.zip`/`*.sql`/`.DS_Store`)、
  配置(`web.config`/`WEB-INF/web.xml`/`.htaccess`)、信息泄露(`phpinfo`/`server-status`)、
  API 文档(`swagger.json`/`openapi.json`/`v2/api-docs`)、**Spring Actuator**
  (`/actuator`/`env`/`heapdump`/`mappings`)、管理后台与源码元数据(`phpmyadmin`/`package.json`/
  `composer.json`/`docker-compose.yml`/`.gitlab-ci.yml`/`.idea`)。约 38 条,各带稳定 `rule_id` +
  严重度 + 命中签名。
- **命中签名 `Sig`**(尽量强,压误报):
  - `BodyAny(&[..])`:2xx 且**解码后**响应体含任一特征串(签名强,天然抗 soft-404)。
  - `Magic(&[u8])`:2xx 且**原始**响应体以魔数开头(zip `PK\x03\x04`、sqlite、heapdump `JAVA`、
    `.DS_Store` `Bud1`)。**刻意不用 `1f 8b`**,以免与「响应被 gzip 压缩」混淆。
  - `Exists`:2xx 且**不同于 soft-404 基线**才算命中。
- **soft-404 基线**:对每个 origin 先请求一个「必不存在」的随机路径,记录其状态码 + 响应体长度;
  `Exists` 类判定时,若探测响应与基线「同码且长度接近(≤64B 或 ≤5%)」即判为软 404 → 不报。
  这是对付「SPA / 自定义 404 对任何路径都回 200」误报的关键。
- `origins(flows)` 去重提炼目标源;`probe_flow(origin, path)` 造带 Host/UA 的 GET 流;
  `evaluate_path(entry, resp, baseline)` 出 `Finding`。

### 2.2 UI(`scanner.rs`,复用主动扫描机制)
- 扫描器页工具条新增 **「敏感文件」** 按钮(Folder 图标);点击 `run_discovery_scan`。
- 后台:每个 origin **先发基线**(不计进度)→ 逐条发高危路径探测 → `evaluate_path`;命中经
  `ScanMsg` 流式回填,与「主动扫描」**共用** `scan_busy`/`scan_ctrl`(暂停/继续/停止)/进度/findings。
- 遵守 `scan_target` 目标选择;`DISCOVERY_ORIGIN_CAP=16` 限制单次最多扫的 origin 数,防狂轰。
- 发现卡片复用既有严重度彩色分组 + 过滤 chip;标题走 i18n 中文。

### 2.3 验证
- `scry_scan` 24 测(含 8 个 discovery:origin 去重 / probe 造头 / BodyAny 命中与漏报 /
  soft-404 压制 / Magic / 无响应 / rule_id 唯一)、`scry_app` 50 测全过;clippy 0 警告(仅上游 `block`)。

## 3. 越权 / 访问控制测试(`scry_scan::authz`,对标 Burp Autorize)
「提权 / 越权」与 SQLi/XSS 不同——它**有状态、要跨身份对比**:同一请求用不同身份重放,看谁
不该拿到却拿到了。

- **内核**(纯函数 + 4 单测):`Identity`(一组鉴权头;`anonymous()` 套用时剥离常见鉴权头)、
  `apply_identity(base, id)`(套身份产出可重放流)、`compare(privileged, test) -> AuthVerdict`
  (`Enforced` 401/403/跳登录 · `Bypass` 低权/匿名拿到与高权相同 2xx 且响应体长度接近 · `Inconclusive`)、
  `evaluate(...)`(匿名命中 = Critical「未授权访问」;低权命中 = High「越权/提权」,覆盖水平 IDOR + 垂直越权)。
- **CLI** `scry_proxy/examples/authz.rs`:`SCRY_ID_HIGH`(基准)/`SCRY_ID_LOW`(可选)+ 总测匿名,
  对一批 URL 多身份重放比对(限速),靶场就绪即可直接跑。GUI 页待 scry_app 可编后接。
- **真站实战修 FP**:对 apizero.cn 带 key 跑主动扫描,唯一「命中」是 `/api/video-parse` url 参数
  在 **JSON** 响应里反射 → 核实为误报。已修 `active.rs`:反射 ≠ XSS,仅 `Content-Type: html`
  才判 High「Reflected XSS」,非 HTML 降 Info「Reflected value (non-HTML)」。

## 4. 后续(同一引擎可继续深化)
- **盲注 SQLi(布尔 + 时间)**:`active.rs` 增不依赖报错回显的注入检测(补齐 sqlmap 那半)。
- 命令注入 / SSRF / 开放重定向 / 更多 XSS 上下文。
- **IDOR id 枚举**:对路径/参数里的数字/uuid id 做 ±1 / 替换,配合 authz 身份判定。
- 内容发现与 **Spider 联动**:爬虫发现的目录作为路径前缀做递归内容发现。
- 高危路径库 / 身份配置可外置(`~/.scry/`)或走扩展系统。
- C2 方向若确有需求,只做「扩展系统外部工具桥」,不进内核。
