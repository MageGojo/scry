# 设计 · 竞品对标与能力缺口分析

> 目的:回答「scry 作为安全渗透/抓包工具,对照业界著名工具还缺什么、哪些功能值得借鉴」。
> 结论:scry 已覆盖 Burp 约 80% 的核心工作台;真正的差距在**渗透深度**与**工作台组织能力**两块,而非基础件。
> 落档日期:2026-06-25。

## 一、scry 现状(已实现,源码核对)

- **内核**:`scry_proxy` 单一 TLS 终止式 MITM —— WebSocket 抓取(只读)、HTTP/2、`upstream` 上游链式、TLS 指纹(JA3/JA4)。
- **四大件**:Proxy(history+拦截改包)/ Repeater(replay)/ Intruder(4 模式+载荷生成器+处理器链+Grep-Extract+并发限速)/ Scanner。
- **漏洞引擎**:`scry_sqli`(报错+布尔+时间盲注,5 方言)、`scry_xss`(上下文感知+WAF 多向量+drission DOM 真执行)、
  `scry_scan`(被动规则 + 主动 SqliError/XssReflect/PathTraversal + Nikto 式 `discovery` 38 路径 + `authz` 越权)。
- **工具页**:Sequencer / Decoder(31 变换+对称加解密) / Comparer / Logger / Extender。
- **扩展系统**:三 Runner(内置 / Python 进程 / WASM 沙箱)+ `HostServices::send_request` 双向 RPC。
- **抓包源**:T1 内置浏览器(过 pinning,免装系统 CA)、T2 托管启动程序、T4 对接代理客户端(sing-box/QX/Proxifier)、被动嗅探+pcapng。
- **配套**:MCP 服务(AI 调度,8 工具)、HAR 导入、复制为 curl/python/js、CA 一键安装+多机共用、会话管理、.app 出包+图标。

## 二、能力缺口(按 ROI 分三档)

### 🔴 第一档 · 渗透杀手锏(与真 Burp 的核心差距)
1. **带外/OOB 交互检测(Collaborator / interactsh)** —— 最大短板。现有引擎全是「基于响应」,打不了盲 SSRF/XXE/SQLi/RCE、盲打 XSS、DNS 外带。需自建或集成 DNS+HTTP 回连服务器。
2. **Session handling / Macros(自动重登 + CSRF/JWT 自动续)** —— authenticated 扫描刚需:session 失效自动跑登录宏、从响应抽 token 带入后续请求。
3. **主动扫描漏洞类扩充** —— 现仅 SqliError/XssReflect/PathTraversal;缺 SSRF/XXE/SSTI/命令注入/CRLF/开放重定向/原型链污染/反序列化(均为「请求→匹配/带外」同构模式)。
4. **nuclei 式 YAML 模板引擎** —— 杠杆最大:扫描规则从硬编码 Rust 改 YAML 模板,直接复用社区几千模板。
5. **SQLi 深化(sqlmap 式)** —— 库/表/字段枚举 dump、文件读写、OS shell、二阶注入、tamper(WAF 绕过)。

### 🟡 第二档 · 工作台组织/体验(对标 Caido / Charles)
6. **Site Map 树状站点地图 + Scope** —— 现为平铺 history+网站分类,缺 host→path 树 + 已测/漏洞标注。
7. **HTTPQL 查询语言** —— 现仅子串搜索;对标 Caido HTTPQL / Burp Bambdas。
8. **Map Local / Mock** —— 现有 Match&Replace,缺「整个响应换本地文件/mock」(Charles 杀手锏)。
9. **WebSocket 改包/重放** —— 现 WS 只抓取,不能拦截改帧/重放(TODO)。
10. **gRPC / Protobuf / GraphQL 解码** —— 现仅 JSON/form。
11. **报告生成(HTML/PDF)** —— findings 不能导出交付。

### 🟢 第三档 · 侦察与扩张(偏主轴,可选/走扩展)
12. 目录/内容高速爆破(ffuf 式:字典管理/递归/自动校准)。
13. 技术栈指纹(Wappalyzer)、请求走私检测(Request Smuggler)、Turbo Intruder 单包竞态。
14. 资产侦察(subfinder/httpx/naabu/Nmap)——建议走扩展系统外接,不内置。
15. Frida 集成(绕 SSL Pinning 抓移动端原生 App)——需关 SIP,场景受限。

## 三、推荐路线(ROI Top 4)
1. **OOB 带外服务器** → 解锁全部盲注。
2. **nuclei YAML 模板引擎** → 借社区之力,扫描能力翻几十倍。
3. **Session/Macros 自动重登** → 让主动能力能打登录站。
4. **Site Map + HTTPQL** → 工作台组织对齐 Caido。

四项均契合现有架构(复用 `scry_proxy::replay` + `scry_scan` + 扩展钩子),不动 MITM 内核,不破坏「纯 Rust / 零环境出包」底线。

## 四、著名工具速查
- 工作台:Burp(Collaborator/Macros/站点地图/Turbo Intruder/Request Smuggler/DOM Invader)、Caido(Rust·HTTPQL/Workflows/Sitemap)、ZAP(YAML Automation/AjaxSpider/被动规则库)、mitmproxy(addons/录制回放/map_local/WireGuard 透明)、Charles+Fiddler(Map Local/Throttle 弱网/Rewrite)、Reqable+HTTP Toolkit(gRPC/扫码连手机/一键拦 app)、Wireshark(协议解析/追踪流/过滤 DSL)。
- 漏洞:nuclei(YAML 模板)、sqlmap(dump/OS shell/tamper)、dalfox/XSStrike(XSS)、Nikto/Wapiti(黑盒)、ffuf/feroxbuster/dirsearch(fuzzing)、jwt_tool。
- 侦察:interactsh(开源 Collaborator)、subfinder/amass(子域)、httpx(探活指纹)、katana(爬虫)、naabu/Nmap(端口)、Wappalyzer(指纹)。
- 利用/动态:Metasploit/CobaltStrike/Sliver(C2,不内置)、Frida/objection(插桩/绕 pinning)。
