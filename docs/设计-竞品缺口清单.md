# Scry 竞品缺口清单(2026-06-25)

> 对标 Burp Suite / Reqable / Charles / Fiddler / mitmproxy / Caido,盘点 scry **当前真实代码**(非文档)还缺什么。
> 已实现的不重复列(见下「已覆盖」)。每项标:现状 · 对标 · ROI · 是否契合 scry 主轴(MITM + replay + 纯函数引擎 + 扩展钩子,尽量不动内核)· 实现思路。

## 第一档:高 ROI、契合现有架构、对标主流招牌 —— ✅ 全部完成(2026-06-25)
> ⑤被动扩规则 → ④报告生成 → ③Param Miner → ①请求走私 → ②竞态,均已落地 + 全绿。

1. ✅ **HTTP 请求走私检测(Request Smuggling)** — 已实现:`scry_proxy::smuggle`(CL.TE / TE.CL 计时法,走底层 socket 发畸形 CL/TE 绕开 `replay` 的 `Connection: close`)+ Scanner 工具条「请求走私」按钮。计时法侵入性、可能误报,finding 标「需人工确认」。
2. ✅ **single-packet attack / 竞态(Turbo Intruder 式)** — 已实现:`scry_proxy::race`(`LastByteSync` 最后字节同步 single-packet + `Parallel` 并行,`tokio::sync::Barrier` 同放扳机,纯函数 `summarize` 判定 + 6 单测)+ 独立 `Tab::Race` 页(并发路数/模式 + 结果表 + 统计卡 + 代理右键「发送到竞态」)。h1 单包已做;真 h2 单包留后续。见 `设计-竞态single-packet.md`。
3. ✅ **Param Miner(隐藏参数挖掘)** — 已实现:`scry_scan::param_miner`(~84 词字典 + 金丝雀反射判定 + URL 回显降权,5 单测)+ Scanner 工具条「参数挖掘」按钮(后台 replay 批量发包,命中进扫描发现 + 报告)。
4. ✅ **报告生成(findings → HTML / Markdown 导出)** — 已实现:`scry_app/src/report.rs` 聚合 Scanner/Authz/Nuclei/XSS/SQLi finding → HTML(彩色徽标 + 统计 + XSS 转义)+ Markdown 导出 `~/.scry/exports/`,Dashboard「导出报告」按钮。
5. ✅ **被动扫描扩规则** — 已实现:`scry_scan::passive` 补 CORS 反射 Origin(回显 Origin + 凭据 High)、混合内容(HTTPS 引 http 子资源)、密钥泄露(AKIA/AIza/ghp_/xoxb-/sk_live_ + PEM 私钥,脱敏)。手写匹配、不引 regex 依赖。

## 第二档:实用、工作台增强 —— ✅ 全部完成(2026-06-25)
> ⑩JWT → ⑥Compose → ⑦GraphQL → ⑨Crawl→Audit → ⑧WS 改帧,均已落地 + 全绿。

6. ✅ **请求构造器 Compose + 环境变量** — 已实现:`Tab::Compose`(`scry_app/src/compose.rs`)从零写请求行 + 头 + body,`{{var}}` 环境变量发送前替换([`substitute_vars`] 纯函数 + 单测),命名集合保存 `~/.scry/compose.json`(载入 / 删除);发包复用 `replay::send`,响应区四视图。代理右键「发送到 Compose」带入流。
7. ✅ **GraphQL 支持** — 已实现:新 crate `scry_graphql`(纯函数,6 测:`minify`/`prettify` 词法美化压缩 + `INTROSPECTION_QUERY` + `parse_introspection` 解析 schema 成类型→字段→参数 + `build_request_body`)+ `Tab::Graphql`(端点 + query/变量/额外头分栏 + 美化/压缩 + 「拉取 Schema」+ schema 浏览树 + 响应)。代理右键「发送到 GraphQL」自动抽 query/variables。
8. ✅ **活动 WebSocket 拦截 / 改帧** — 已实现(**动了内核,默认透传零行为变化**):`scry_proxy::websocket` 加 `WsRewriteRule`/`WsRuleDir`/`rewrite_text`/`encode_frame`(纯函数 + 单测);`ProxyConfig.ws_rewrite: Option<Arc<Vec<WsRewriteRule>>>`;`mitm::handle_websocket` 按方向选 `pump_ws`(透传)或新增 `pump_ws_rewrite`(解帧 → 完整文本帧字面量改写 → 重编码按方向 mask → 转发 + 记录改写后内容)。规则为空 = 老的字节透传。UI 在 Proxy → Options 加「WebSocket 改帧」卡(方向 + find → replace + 删除),持久化 `~/.scry/ws_rules.json`,抓包启动时注入。定界:只改**完整文本帧**(分片/二进制/控制帧原样转发),仅 find→replace(丢帧未做)。
9. ✅ **Crawl → Audit 流水线联动** — 已实现:Spider 页加「爬完自动审计」开关(`crawl_then_audit`);爬完(`drain_crawl_results` 见 `done`)自动把扫描范围限到爬过的 host(`crawl_audit_host`)、跳到扫描器、跑被动 + 主动扫描(复用 `run_passive_scan`/`run_active_scan`)。爬虫流量本就经 MITM 落历史,这里只做编排闭环。
10. ✅ **JWT 攻击套件进 GUI** — 已实现:新 crate `scry_jwt`(纯函数,9 测:b64url/HMAC-SHA256/`decode`/`forge_none`(含大小写变体)/`sign_hs256`/`sign_with_header`/`forge_kid`/`verify_hs256`/`crack_hs256` + 内置弱密钥表)+ `Tab::Jwt`(粘贴令牌实时解码 + alg 徽标 + payload 编辑 + alg:none / HS256 签 / kid 注入 / 弱密钥爆破 + 结果复制)。代理右键「发送到 JWT」从请求(Authorization/Cookie/body)自动提取首个 JWT。

## 第三档:专业方向 / 大件(按需) —— 部分完成(2026-06-25)
11. ✅(响应可判定子集)**主动扫描补漏洞类型** — 已实现:`scry_scan::active` 新增 4 个 ProbeKind(**零改 app**,新 finding 自动显示)—— **LDAP 注入**(报错型,过滤器括号破坏 → LDAP 异常签名)、**XPath 注入**(报错型,引号破坏 → XPath 异常签名)、**SSI 注入**(`<!--#exec-->` 被执行 = marker 出现且 `#exec` 被消费,排除原样/转义回显)、**主机头注入/投毒**(Host + X-Forwarded-Host 指向唯一标记域,回显进 body/Location 即命中)。**定界**:CSTI / 客户端原型链污染(需浏览器执行)、不安全反序列化 / Web 缓存投毒(需带外 / 多请求关联)**未做**——纯响应判定不可靠,留给 OOB / 浏览器路径。
12. **OAuth / SAML / OIDC 测试** — 现状:无。ROI:中低(专业)。大件,**未做**。
13. **gRPC(带 .proto 结构化)/ 真 HTTP/3 MITM(quinn 终止式)** — 现状:gRPC 仅无 schema 解码、HTTP/3 仅被动。ROI:低、工程量大。**未做**。
14. ✅(图表部分)**流量统计 / 图表 / 工程文件** — 已实现:`Tab::Stats`(`scry_app/src/stats.rs`)对当前会话流量纯函数聚合(`aggregate` + `categorize_content_type`,3 单测)+ 概览指标卡(总数 / HTTPS / HTTP / WS / 响应字节)+ 横向条形图(按方法 / 状态码段 / 类型 / Top 主机)。**定界**:`.scry` 工程文件(保存 / 恢复全工作区)**未做**。

## 明确不做(与定位冲突)
- 系统代理一键开关 / 反向代理(scry 坚持不抢系统代理)。
- 托管式 Collaborator 服务器(用公共/自建 interactsh 即可,已由 `scry_oob` 覆盖客户端)。

## 建议
**第一档 + 第二档已全部完成**。剩余只有**第三档**(专业方向 / 大件,按需):主动扫描补漏洞类型(LDAP/XPath/SSI/缓存投毒/原型污染/反序列化/主机头)、OAuth/SAML/OIDC、gRPC 带 .proto / 真 HTTP/3 MITM(quinn)、流量统计图表 / `.scry` 工程文件。这些 ROI 分散或工程量大,待用户点选再做。
