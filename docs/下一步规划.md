## #1 主动扫描漏洞类扩充 ✅ 已完成(2026-06-25)

**落点**:`scry_scan::active`(纯函数内核)。`scanner.rs` 的 `run_active_scan` 只遍历 `generate_probes` + 对每个 `evaluate`,故**零改 scry_app**(新 finding 自动显示;中文标题 i18n 可选补)。

新增 `ProbeKind`(均为「基于响应」的检测,不依赖 OOB):
- **SSTI**:polyglot `{{1337*1337}}${1337*1337}` → 命中独特乘积 `1787569`(几乎不可能自然出现)= 模板被求值。Critical。
- **命令注入(回显型)**:`;id;` → 命中 `uid=`+`gid=`(id 命令特征)。Critical。
- **CRLF / 响应头注入**:`\r\nX-Scry-Crlf: scry1337` → 命中响应头里出现该独特头。Medium。
- **开放重定向**:仅疑似跳转参数(url/redirect/next/return/dest/target…)注入独特外部域 → 命中 3xx + Location 指向它。Medium。
- **SSRF 云元数据**:仅疑似 URL 参数注入 `http://169.254.169.254/latest/meta-data/` → 命中 ≥2 个 IMDS 特征(ami-id/instance-id/iam/…)。Critical。
- **XXE(回显型)**:对 XML 请求体替换为外部实体读 `/etc/passwd` → 命中 `root:x:0:0`。High。

**盲检测留给 #2 OOB**:盲 SSRF/XXE/RCE/SQLi、盲打 XSS 在有了 OOB 后,把上面这些 payload 的「回连 scry OOB 域名」变体接上即可。

**验证(已完成)**:`scry_scan` **37 测全过**(+8 新)、`clippy --all-targets` **0 警告**、`scry_app` 编译通过;i18n 补 6 条中文标题;**零改 scanner.rs**(自动用新探测)。后续靶场实打。
