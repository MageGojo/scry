# 设计 · 代理拦截规则(自定义拦截 + Match & Replace + 右键)

> 目标(用户):在代理页加「右键拦截」入口,支持**自定义拦截**(按条件只拦匹配的流量),支持**主流的拦截修改方式**(尤其 Match & Replace 自动改包)。

## 现状(已有)
- 交互式拦截已完成:Intercept 页两个开关(拦请求 / 拦响应)+ 暂停队列 + 编辑原始报文 + 放行 / 丢弃 / 全放行。
- 引擎在 `ExtRegistry::on_request/on_response` 钩子里调 `maybe_intercept`,命中即把报文发 UI 并**阻塞**等用户决策(`scry_app/src/{ext,intercept}.rs`,不改 `scry_proxy`)。
- 短板:拦截是**全有全无**(开了就拦该方向所有流量);除「手动编辑暂停的报文」外没有自动改包手段;没有右键快捷入口。

## 本次新增(三块)

### 1. 自定义拦截范围(Scope)
- 数据:`ScopeRule { enabled, dir(请求/响应), field, op, value, negate, intercept(拦截/跳过) }`。
- 字段 `Field`:URL / 主机 / 路径 / 方法 / 状态码 / 请求头 / 请求体 / 响应头 / 响应体 / 任意。
- 算子 `Op`:包含 / 等于 / 通配(`*`) / 正则(`regex` crate)。
- 求值 `should_intercept(flow, dir, rules, default_on)`(只在该方向开关已开时调用,`default_on=true`):
  - 该方向**无规则** → 拦全部(保持旧行为)。
  - 命中任一「跳过」规则 → **不拦**(排除优先)。
  - 存在「拦截」规则 → 命中其一才拦;否则不拦。
  - 只有「跳过」规则且都没命中 → 拦。

### 2. Match & Replace 替换规则(主流自动改包)
- 数据:`ReplaceRule { enabled, target, is_regex, find, replace }`。
- 目标 `Target`:请求路径 / 请求头 / 请求体 / 响应头 / 响应体(覆盖 Burp 最常用 4+1 种)。
- 应用(自动,无需暂停):`on_request` 应用请求向规则、`on_response` 应用响应向规则。
  - 头:对每行 `Name: Value` 做替换后回填(`find` 为空 = 追加 `replace` 作为新头)。
  - 体 / 路径:整体字符串替换(`is_regex` 用 `Regex::replace_all`,否则字面 `str::replace`)。
- `wants_response_hook()` 增条件:存在启用的响应向替换规则时也要让引擎重建响应字节。

### 3. 右键入口(代理历史行)
- 「拦截此主机的请求 / 响应」→ 加一条 `Host 等于 <host>` 的拦截 Scope 规则并打开对应开关 + 跳到 Intercept 页。
- 「不拦截此主机」→ 加一条跳过 Scope 规则(排除噪声)。
- 「添加替换规则…」→ 跳到 Options 页规则编辑器。

## 落点
- 纯逻辑 + 单测:`scry_app/src/rules.rs`(`Field/Op/Condition/ScopeRule/ReplaceRule` + 编译 + `matches`/`apply`/`should_intercept`)。
- 引擎接入:`ext.rs` `ExtRegistry` 加 `Mutex<Vec<CompiledScope>>` / `Mutex<Vec<CompiledReplace>>` + setter;`on_request/on_response` 先应用替换再 `maybe_intercept`(用 scope);`set_*` 由 UI 改规则时调用,抓包启动时一并推送。
- UI:**Options 页**(原为空)做规则编辑器(拦截范围 + 替换规则两卡:只读规则行 + 底部添加表单)。Intercept 页保留实时队列与开关。
- 右键:`chrome.rs` 上下文菜单加分组。
- i18n:补中英文键。

## 取舍
- 规则的"真相源"放 UI 侧 `ScryApp` 状态(纯结构、易编辑);每次增删 / 启停或抓包启动时,把**编译快照**(预编译正则)推给 `ExtRegistry`,引擎逐请求只读快照,避免逐请求编译正则。
- v1 规则编辑为「列表 + 增删 + 启停」,不做行内编辑(改 = 删了重加);足够覆盖主流用法,UI 复杂度可控(契合 gpui 输入框需预建 Entity 的约束)。
- 正则非法时该条不命中(不 panic),UI 后续可加校验红字。
