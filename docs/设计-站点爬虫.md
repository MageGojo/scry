# 设计 · 站点爬虫(Spider / Crawler)—— 浏览器驱动(drission / CDP)

对标 Burp 的 **Crawl**:从种子 URL 出发,用**真实浏览器**(无头 Chrome,经 CDP 驱动)访问页面 →
渲染并执行 JS → 解析渲染后 HTML 抽链接 → 同站 **BFS** 递归,边走边把页面流量落进历史(可随后被扫描)。

> **为何从「协议级」改为「浏览器驱动」**(本轮重构):协议级爬虫(直接发 `GET`)遇到**加密参数 /
> 签名 / SPA 动态渲染**就得**逐站逆向**,做不到"通用、不针对某个网站"。真实浏览器直接执行页面 JS,
> 拿到的是渲染后的 DOM 与真实发出的请求,天生通用。代价是更重(每页起真实渲染)。

## 引擎:drission(Rust 版 DrissionPage)

- crate **`drission`**(默认 `cdp` feature):驱动本机 **Google Chrome** / Edge / Brave / Chromium(CDP),
  找不到时自动下载 Chrome for Testing。scry 以 path 依赖 `../drission-rs`(workspace 依赖)。
- 关键 API:
  - `ChromiumBrowser::launch(ChromiumOptions::new().headless(true).proxy(..).add_arg(..))`
  - `browser.new_tab(None)` → `ChromiumTab`
  - `tab.get(url) -> bool`、`tab.wait_loaded()`、`tab.html()`(渲染后 DOM)、`browser.quit()`

## 结果汇入「代理」历史(复用 MITM,不另存)

无头 Chrome 的启动参数让它把流量喂给 scry 的 MITM 内核:

- `--proxy-server=http://127.0.0.1:8888` —— 指向 scry MITM(`ChromiumOptions::proxy`);
- `--ignore-certificate-errors-spki-list=<scry CA 的 SPKI>`(`scry_ca::spki_sha256_base64()`,经
  `add_arg` 注入)—— **免装系统 CA、连 pinning 都过**(配合 drission 默认的独立临时 user-data-dir 才生效)。

于是浏览器**全部流量**(主文档 + JS/CSS/img + XHR)经 MITM **自动解密落库**,由抓包推流进
「代理」页 HTTP 历史(可被扫描器扫)。**爬虫本身不再手动构造 / 落库 `HttpFlow`**(那由 MITM 接管)——
这正是 Burp「内置浏览器」抓包的本质,本爬虫等于它的自动化版。

## 分层(纯函数内核 + 浏览器 runner)

- **`scry_crawl`(内核 crate,纯函数 + 13 单测)** —— **保留复用**:`extract_links(base, html)` +
  `Crawler` BFS 调度器(`new`/`next_target`/`feed`,同站 / 深度 / 页数上限 / 去重)+ `CrawlConfig`。
  唯一变化:喂给 `feed` 的 HTML 从"协议 `GET` 的原始响应体"换成"**浏览器渲染后的 DOM**(`tab.html()`)",
  于是 **JS 动态生成的链接也能被发现**。

- **`scry_app::crawler`(浏览器 runner)**:在**独立 OS 线程**上起 **多线程** tokio runtime `block_on`
  串行驱动 CDP(CDP 客户端需后台读循环 → 多线程 runtime 最稳;不同于 Scanner/Repeater 的 current-thread)。
  - 启动前:`ensure_proxy_running` 确保 MITM(8888)在跑;`scry_ca` 算 CA SPKI;优先用系统已装 Chrome
    (`launcher::find_browser`),否则交 drission 自动下载。
  - 每页:`tab.get(url)` 导航(流量经 MITM 落历史)→ `wait_loaded` + 静置 `SETTLE_MS=700`(给 JS/XHR
    渲染、生成动态链接的时间;CDP 后端 `wait()` 无 `secs`,用 `tokio::time::sleep`)→ `tab.html()`
    喂回内核抽链接 → BFS 取下一目标。
  - 经 `mpsc<CrawlMsg>` 流式回传进度 + 本次访问的 URL,前台 150ms 轮询 `drain_crawl_results` 并入
    **「本次发现列表」**(`crawl_visited`)+ 刷进度。停止 = 置 `CRAWL_STOP` 位 + 丢接收端 + `browser.quit()`。

## 对标 Burp:全站发现 + 浏览器生命周期(本轮增强)

**发现不止 `<a href>`**(解决"未在首页链接的页也能抓到",如孤立博客页):
- **robots.txt + sitemap.xml**:开爬先经 scry `replay` 拉每个种子 origin 的 `/robots.txt`(取 `Sitemap:`
  与 `Allow/Disallow` 路径)+ `/sitemap.xml`(取 `<loc>`;sitemap index 再下钻一层),`Crawler::enqueue`
  注入队列。**这是 Burp 发现孤立页的主要手段**。
- **渲染后 DOM**:真实浏览器执行 JS 后才 `tab.html()`,动态生成的链接也在。
- **扫 JS / 内联文本**:`scry_crawl::extract_urls_from_text` 从脚本字符串里再抠 `"/path"` 与 http(s) URL
  (很多站把路由 / API / 文章地址写在 JS 里,DOM 的 `<a>` 抓不到);`feed` 合并属性链接 + 文本链接。
- 子资源 / XHR(浏览器加载每页时实际请求的一切)经 MITM **已全部进代理历史**(被动覆盖"任何资源")。

**浏览器生命周期受控**(修"爬完 / 关软件 Chrome 残留"):改 **connect 模式**——由 scry 用 `launcher::
launch_headless_browser_debug` 自启无头 Chrome(`--headless=new` + 空闲 CDP 调试端口 + proxy 8888 +
SPKI + 独立 `spider-profile`),drission `ChromiumBrowser::connect` 接管驱动;Chrome 进程句柄存
`state.crawl_child`,**停止 / 爬完 / 关软件(`Drop for ScryApp`)都统一 `kill_crawl_browser`**,杜绝孤儿。

## UI:独立「爬虫」页签(对齐 Burp Crawl)

- 顶栏新增 `Tab::Spider`「爬虫」(图标 `GitBranch`),左栏 Tools 也加入口。**入口与发现列表同处一页**,
  不再像旧版那样把爬虫工具条塞进「扫描器」页(那时"入口在扫描器、结果在代理"的割裂感是本轮改的起因)。
- 页内:标题 + 一句话说明(浏览器驱动 · 流量汇入代理历史)+ 爬取工具条(种子 / 深度 1–5 /
  页数 20·40·60·100·200·500 / 爬取·停止 / 进度)+ **本次发现列表**(逐页:成功绿点 / 失败红点 + URL)。
- 闭环:爬取站点 → 报文进「代理」历史(Proxy 页可见)→ 扫描器选该 host → 被动/主动扫描。

## 取舍与边界(诚实记录)

- **重**:每页起真实渲染 + 静置等待,比协议级慢;换来通用性(吃 JS/SPA、绕开加密参数 / 签名逆向)。
- 只抓 scry 自启的这个无头浏览器的流量(经 MITM)——与 Burp 内置浏览器同理;抓"已运行的任意 App"仍走 T4。
- 首次若本机无任何 Chromium 系浏览器,drission 会自动下载 Chrome for Testing(一次性,需网络)。
- 遇站点 pinning:SPKI 白名单已覆盖 scry CA;真正的第三方 pinning / 微信 MMTLS 仍解不开(任何 MITM 同理)。
- 当前单标签串行 BFS;高并发(drission `ChromiumPool`)与 robots.txt / 表单深抓为后续增强。
