# 设计 · 扫描引擎与高级能力(四特性)

> 本轮目标:被动/主动**扫描引擎**、**TLS 指纹伪装**、底层 **L2/L3 pcapng** 导出、**零环境出包**。
> 组件化:能纯函数化的(规则判定 / 探测生成 / pcapng 编码 / TLS profile 映射)全部做成可单测的纯函数,
> IO / 网络 / 打包放最薄的外层。

## 1. 扫描引擎(`scry_scan` crate)

### 被动扫描(passive,纯函数,只读 `HttpFlow`)
对每条已抓到的流跑一组规则,产出 `Finding{ rule_id, title, severity, url, detail }`。规则(MVP):
- 安全响应头缺失:HSTS(HTTPS)、`X-Content-Type-Options`、`X-Frame-Options`/CSP frame-ancestors、CSP。
- Cookie 缺旗标:`Secure`(HTTPS 下)、`HttpOnly`、`SameSite`;**HTTP 明文下发会话 Cookie**(高危)。
- CORS 配置:`Access-Control-Allow-Origin: *` 且 `Allow-Credentials: true`(高危)。
- 信息泄露:`Server` / `X-Powered-By` 版本号(info)。
- URL 敏感参数:`password=` / `token=` / `api_key=` / `secret=` 出现在查询串(中危)。
- 明文凭证:HTTP 上的 `Authorization: Basic`(高危)。
- 反射输入:某请求参数值**原样**出现在 HTML 响应体(潜在 XSS,中危)。
- 错误泄露:响应体含 SQL 报错 / 异常栈关键字(中危)。
- 目录列表:HTML 含「Index of /」(中危)。

`scan_flows(&[HttpFlow])`:对全部流跑规则 → 按 `(rule_id, url)` 去重 → 按严重度降序排序。

### 主动扫描(active,探测生成纯函数 + 命中判定纯函数;发送复用 `scry_proxy::replay`)
- `generate_probes(base)`:对每个查询参数注入 payload,克隆出**变异请求流**(不发送):
  - SQLi(error-based):值后追加 `'`;命中判定 = 响应体出现 SQL 报错串。
  - 反射 XSS:值替换为唯一 marker `scry<svg/onload>`;命中 = marker 原样回显。
  - 路径穿越:值替换为 `../../../../etc/passwd`;命中 = 响应体出现 `root:x:0:0`。
- `evaluate(probe, resp_flow)`:对探测响应做特征匹配 → `Option<Finding>`。
- UI 侧 runner(后续):把变异流逐个丢给 `replay::send`(后台 tokio),回主线程汇总 findings。

### UI:Scanner 页签
顶栏加「扫描器」页;一键「被动扫描」当前 flows → findings 列表(严重度彩色徽标 + 标题 + URL + 详情),
按严重度分组计数(Critical/High/Medium/Low/Info)。主动扫描按钮后续接 runner。

## 2. TLS 指纹伪装(`scry_proxy::tls_profile` + `scry_proxy::fingerprint`)
MITM 上游(scry → 目标)用 rustls;默认 JA3/JA4 易被识别为 rustls。提供**浏览器 profile**,在 rustls 可控范围内贴近:
- **可控并已做**:ALPN(`h2,http/1.1` vs 仅 `http/1.1`)、**密码套件顺序**、**椭圆曲线(supported_groups)顺序**(均经自定义 `CryptoProvider`)。
- profile:Chrome / Firefox / Safari / curl / Default;`provider_for(profile) -> CryptoProvider` 做成纯映射 + 单测;进程级 `set_active/active` 让 `build_client_config` 免改签名读取当前档。
- **单一构造入口**:`mitm::build_client_config_for(profile)` 同时供 MITM 上游 / Repeater 重放 / 指纹计算共用 → 显示的指纹与线上握手必然一致。

### 2.1 指纹可视化(`fingerprint.rs`,JA3 + JA4,不靠猜)
让 rustls **真吐 ClientHello**(`ClientConnection::write_tls` 写进内存 buffer,不连网、不握手),再按规范从**真实字节**解析五段算 JA3,以及算 JA4(TLS,TCP)。UI 设置页展示当前档的 **JA4**。
- **关键发现(务必记住)**:rustls 0.23 **每次握手都随机化 ClientHello 扩展顺序**(抗固化,类似现代浏览器)。
  → JA3 的「扩展段」逐连接变化、**JA3 本身不稳定**;而 **JA4 对密码 / 扩展排序后再哈希,稳定**。故 UI 以 **JA4 为准**,JA3 仅作单次采样参考。
- **本套伪装的真实效果(诚实)**:只改 rustls 可控的密码 / 曲线**顺序** + ALPN;JA4 故意忽略顺序 →
  各档 JA4 仅 ALPN 段有别(Chrome/FF/Safari=`...h2`,Default/curl=`...h1`),JA4_b/JA4_c 因排序而完全相同。
  实测 Chrome 档 JA4 ≈ `t13d1011h2_61a7ad8aa9b6_3fcd1a44f3e3`(可拿去 tls.peet.ws 对照)。
- **局限与后续**:字节级 JA3/JA4 真正等于浏览器(套件集合差异、扩展顺序 / GREASE / `application_settings` 等、精确布局)rustls 做不到,
  需 BoringSSL(`boring` / `rquest`),工程重,列为后续。开发工具:`cargo run -p scry_proxy --example ja3` 打印各档真实 JA4/JA3。

## 3. 底层 L2/L3 pcapng 导出(`scry_sniff::pcapng`)
被动嗅探时把**原始链路帧**写成标准 **pcapng**(Wireshark 可直接打开),保留 L2(以太网)/L3(IP)。
- 编码器(纯函数 + 单测):SHB(Section Header Block)+ IDB(Interface Description Block,带 linktype)+
  每包 EPB(Enhanced Packet Block,含时间戳 / 原始字节);小端,4 字节对齐,块长前后一致。
- `PcapngWriter`:`new(path, linktype)` 写 SHB+IDB;`write_packet(ts, bytes)` 追加 EPB;Drop/flush 落盘。
- `run` 增可选 `pcapng_out`:每个抓到的帧在 HTTP 重组之外**同时**落一份 pcapng(save-first 不变)。
- 单测:写一两个包 → 读回校验魔数(0x0A0D0D0A)/ byte-order magic(0x1A2B3C4D)/ 块长一致。

## 4. 零环境出包(macOS `.app`)
gpui 程序要双击即用需打成 `.app` bundle。
- `scripts/build_mac.sh`:`cargo build --release -p scry_app` → 造 `Scry.app/Contents/{MacOS,Resources}`,
  拷二进制、写 `Info.plist`(bundle id / 名称 / 高分屏 `NSHighResolutionCapable`)、放图标(有则用)。
- 资源自带:把根 CA 生成逻辑保持运行时在 `~/.scry` 落地(无需打包);如需图标后补 `.icns`。
- release profile 已是体积优化(`opt-level=z`+lto+strip,不 panic=abort)。
- 验证:`ditto`/`zip` 出可分发压缩包;在干净目录双击 `.app` 起窗(本机验证)。
- 说明:macOS 纯 Rust+gpui 不依赖 VC++ 运行时(那是 Windows 的事);只需 `.app` 结构 + 可选签名/公证。
